## Post-submission reflection

1. Describe your problem-solving approach while building this capstone.What steps did you take to understand the requirements, plan your solution, and implement the features?
> My problem solving appproach was to start by outlining my project and grouping like features together. I then when and made sure that the pages where properly routed to each other before working on the functionality of each page, working on one feature at a time. I then used the provided tests as a way to gauge my design and its functionality, tweaking my code as needed.

2. Explain one key technical decision you made during your implementation.Why did you choose this approach over other possible solutions?
> One key technical decision I made was to resuse the CardForm component. As opposed to having separate forms for both Edit Card and Add Card, resuing the CardForm component allowed me reduce lines of code in other components.

3. AI Use Disclosure
> No AI was used, I relied on the previous lessons and examples throughout the course.

4. What is one area of frontend engineering that you want to strengthen based on your experience developing this project?
> I would really like to get more comfortable using redux. While I understand basic state principles, for a project even of this smaller scale it was a little bit of a hassle making sure that the state worked as intended. Hopefully in the future I can experiment more with redux to feel confident on using it in a personal project.
